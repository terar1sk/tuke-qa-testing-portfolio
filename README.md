# 🔍 TUKE Website - Independent QA Audit Portfolio

## 📢 Project Disclaimer
This repository is a **personal training playground**. It is NOT an official university assignment or a professional audit. I am using the `tuke.sk` portal as a real-world environment to practice and refine my **Quality Assurance skills**, learn how to document findings, and explore web technologies from a tester's perspective.

---
## 🚀 Overview
The main goal of this project is to move from theory to practice. By analyzing a live, complex system like the Technical University of Košice portal, I am developing a "tester's mindset" and learning how to communicate technical issues effectively.

### What I am practicing here:
- **General QA Methodologies:** Learning how to approach a new product and where to look for potential improvements.
- **Defect Identification:** Finding common web issues, from simple UI glitches to console warnings.
- **Documentation Standards:** Writing clear, professional Bug Reports and Test Cases that a developer could easily understand.
- **Tool Mastery:** Getting hands-on experience with Browser DevTools (Console, Network, Elements) and analysis tools.

---
## 🧩 Practice Areas
I've divided my practice into several sections to cover different aspects of testing:
- **UI & Content:** Checking how the site looks on different screens and how the language switching works.
- **Functional Testing:** Verifying that buttons, links, and search forms behave as expected.
- **Technical Observation:** Identifying errors or warnings in the developer console to understand what's happening "under the hood."

## 🛠 Tools Used
- **Manual Testing:** Using a Black Box approach to verify functionality and user experience.
- **Zen Browser:** A Gecko-based browser (Firefox engine). Used to verify site behavior and script execution on a non-Chromium architecture.
- **DevTools:** Utilizing Inspector, Console, and Network tabs for real-time system monitoring.
- **Markdown:** A simple way to format text. I use it to make my bug reports and project descriptions look organized on GitHub.
